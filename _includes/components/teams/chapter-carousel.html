Frontend

Our Frontend is built as a Single Page Application with React and TypeScript and is bundled via Webpack and Babel.
We make heavy use of Material-UI components and our Dialogue Builder, where our customers edit the conversation flow, is built with D3.js.
We talk to the backend APIs via REST with the help of Axios and use OpenAPI specs to make sure we understand each other. We currently hold our data in Redux but are currently preparing the move towards GraphQL and Apollo which will replace our REST and Redux implementation soon.

Backend

Our backend chapter believes in micro-services allowing us to scale up our system as freely as possible. The main flavor of our services in Node.Js and Typescript. For our persistence storage we chose MongoDB and use Elasticsearch to search through large volumes of data quickly.
All services communicate with HTTP requests following REST practices and we are working on introducing a GraphQL API gateway.
Leveraging real time streaming to ElasticSearch, aggregate caches and future GraphQL API gateway we are always on a lookout for performance increase.

Machine Learning

We use the Tensorflow machine learning framework for creating AI models and the model trainings are performed on our own model training system utilising dynamically provisioned Google Cloud VMs and Redis queues. This allows us to run concurrent AI model trainings without tackling scalability issues.
In addition to that, we have many Python-based micro-services using Tornado and Flask frameworks.
Besides essential AI functions like training and inferencing, our machine learning stack includes auxiliary functions for search where we make use of ElasticSearch.

DevOps

Our Platform Engineering team strives towards improving efficiency and resiliency of the platform by leveraging Infrastructure as code, creating self service platforms and eliminating toil. We help developers build resilient, robust and highly scalable micro-services and deliver infrastructure for enabling highly reliable applications and rapid development.
All of our workloads are containerized and we run them on top of our swarm clusters hosted on Google Cloud. Our CI/CD pipelines run on bitbucket. We use ELK stack for logging and grafana/moira for monitoring and alerts. We are now actively migrating to kubernetes.

AI Research

AI research uses a wide variety of technologies, libraries and models to find the best solution for our business problems. The core tech stack is the same as in machine learning (Tensorflow, Python data science stack) and part of the code base used for our experiments is the actual production code.
The team is not limited in what technologies to explore, instead we are interested in the results and in getting our hypotheses tested quickly. We decide on technologies independently but also discuss them in the team and with machine learning engineers to ensure continuity in research and compatibility of the production implementation.
